Project

NoiseAware

UX Audit, User Research, & Redesign

Background

NoiseAware is a privacy-safe smart home solution that protects rental properties by monitoring noise levels. The service alerts the host and guest through a web or mobile experience when the noise levels go over a set threshold, allowing property managers to proactively address potential noise violations before they become a significant problem.

Hired to guide the product team in building a better user experience, I audited the full user journey, beginning with marketing/customer acquisition. The audit exposed inconsistencies in design language, weak UX writing, and a complex experience that was the result of applying endless hot-fixes to address customer requests.

Research & Discovery

After implementing some best practices, I was hired full-time to lead an effort to improve the product’s user experience and establish a more sustainable process for designing the product. The first task was a deep dive into the user needs across a wide range of customer segments. To address the seven different user personas within four market segments, we conducted user interviews to uncover common themes between the small-scale managers and large-scale operations that were utilizing the product.

User Flows

Having a common understanding of each user’s journey is an important step in designing product. While there are different levels of complexity across different markets, the core functionality of the NoiseAware is universal: set a threshold and trigger an alert when the threshold is reached. With this in mind, we redesigned the mechanism for setting a threshold as well as the alert flow, pulling insights from the user research and testing with our user base as we designed the new interactions.
