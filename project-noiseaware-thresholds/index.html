Project

NoiseAware

UX Audit, User Research, & Redesign

Background

NoiseAware is a privacy-safe smart home solution that protects rental properties by monitoring noise levels. The service alerts the host and guest through a web or mobile experience when the noise levels go over a set threshold, allowing property managers to proactively address potential noise violations before they become a significant problem.

Research & Discovery

After implementing some best practices during my contract with Noiseaware, I was hired full-time to lead an effort to improve the product’s user experience and establish a more sustainable process for designing the product. The first task was a deep dive into the user needs across a wide range of customer segments. To address the seven different user personas within four market segments, we conducted user interviews to uncover common themes between the small-scale managers and large-scale operations that were utilizing the product.

User Flows

Establishing a common understanding of each user’s journey would be an important step in redesigning the product as a team.

While there are different levels of complexity across different markets, the core functionality of the NoiseAware is universal: set a threshold and trigger an alert when the threshold is reached. With this in mind, we redesigned the mechanism for setting a threshold as well as the alert flow, pulling insights from the user research and testing with our user base as we designed the new interactions.

Threshold Management: Solution

A highly requested time-saving feature was designed: the capability to create user profiles, enabling batch updates of thresholds across multiple properties. While designing this feature, we also overhauled the custom threshold mechanism.

These were crucial timesaving improvements that would serve both the customer and, in turn, the customer support team, as threshold management was the #1 reason for customer support calls. Moving away from database-driven drop-downs and brand-specific language, we designed a simple and easy-to-understand scheduling tool that used modern mechanisms to adjust thresholds.
