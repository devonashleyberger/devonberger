ReCharge Payments

“Subscription software to grow sales through recurring Shopify customers”

Started as “3 guys in Mike’s apartment”, ReCharge was using Bootstrap styling and the founders’ intuition to guide product development. I joined the team to bring consistency and intention to the user experience, while elevating the identity of the product within the market.

DESIGN LEAD

2016–2017

USER EXPERIENCE, USER INTERFACE & IDENTITY

ReCharge’s customer acquisition strategy relied heavily on Shopify’s review system. This resulted in a short-term, user-driven feature roadmap. To keep customer service requests down, the product experience needed to be holistically debugged. The scalability of the product would improve exponentially with the refactor, but first we needed to level set our understanding of UX or risk misalignment of expectations.

Product Patterns

As the first design hire, I tackled an interface that directly reflected the structure of the database. To reduce potential strain on the back-end team while improving the user flow, the front-end team and I partnered in refactoring UI patterns and language on the client side.

Implementation

Subscription Rules

A “ruleset” is the merchant’s primary tool for setting up subscriptions. By starting the refactor here, we were able to audit and update the most important language used throughout the user experience.

Customer Portal

Although the customer is the primary user of the Customer Portal, merchants need to access the same data (and in some permission configurations, more data) to troubleshoot any customer service issues that might come up. In the refactor, hierarchy was brought to a permission-based data view that would be applicable to both user types.

USERS

CUSTOMERS, SHOPIFY MERCHANTS

Merchant Dashboard

Once a ruleset has been set up and launched; the merchant needs the ability to manage their customer data, gain insight into churn, and possibly add integrations that might improve customer retention.
