USER EXPERIENCE, USER INTERFACE, and IDENTITY

ReCharge’s customer acquisition strategy relied heavily on Shopify’s review system. This resulted in a short-term, user-driven feature roadmap. To keep customer service requests down, the product experience needed to be holistically debugged.

Product Patterns

ReCharge was using Bootstrap styling and the founders’ intuition to guide the product. I joined the team to bring consistency and intention to the user experience, while elevating the identity of the product within the market.

To reduce potential strain on the back-end team while improving the user flow, the front-end team and I partnered in refactoring UI patterns and language on the client side.

Pattern Implementation

Subscription Rules

USERS: SHOPIFY MERCHANTS, SHOPIFY PARTNERS

A “ruleset” is the merchant’s primary tool for setting up subscriptions. By starting the refactor here, we were able to audit and update the most important language used throughout the user experience.

Pattern Implementation

Customer Portal

USERS: CUSTOMERS, SHOPIFY MERCHANTS

Although the customer is the primary user of the Customer Portal, merchants need to access the same data (and in some permission configurations, more data) to troubleshoot any customer service issues that might come up. In the refactor, a view applicable to both user workflows was implemented.
